# Machine-Translation
利用transformer结构实现英文-中文的翻译
这里使用MarianMT预训练模型来对数据集进行分词
通过全参数微调和LORA在部分投影层加入低秩矩阵来减少训练参数的两种方法对模型进行微调

不同tansformer结构的不同:
模型：BART,T5,MarianMT                BERT,ROBEKTA                                  GPT,LLAMA,ChatGLM
类型：生成型                          理解型                                         创作型
结构：Encoder-Decoder                 Encoder-Only(双向编码，看上下文进行预测)       Decoder-Only(单向/因果注意力)
功能：序列-序列，翻译，摘要，问答      完形填空，情感分析，问答，句子相似度           下一词预测，文本生成，对话，代码生成
输出：可变长向量                       输出固定长度向量                              可变长向量

分词：将文本映射成ID，词表可以随意生成这是一个纯统计过程，模型训练则是用来学习语义，故只要分词一致模型就能学会对应向量表示
子词：并非语言学概念，只是训练妥协的产物，拆分成字符则学习效率过低(词表过小)，拆分成单词则参数爆炸容易过拟合(词表过大)
      翻译质量并不依赖与分词质量而是依赖与向量表示质量(模型翻译本质上就是数字序列到数字序列的映射，并非语言学上的概念)
      拆分子词主要是来解决未登录词和控制序列长度(序列越短注意力计算越快O(n^2))
 Decoder之所以可以生成可变长向量主要归功与交叉注意力机制，其机制为自回归生成
 对于transformer结构的详细实现以及各部件功能在model.py中有详细介绍，对于模型训练的抽象过程在training_process.py中做了可视化演示
